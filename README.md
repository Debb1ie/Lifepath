# LifePath

**A life problem solver platform.** Describe any real situation — career, school, money, burnout, decisions — and get a personalized AI roadmap, a visual path diagram, and a skill framework. Not another blog post. Not generic advice. Structured guidance built for your specific situation.

---

## What's Inside

| Page | What it does |
|---|---|
| **Home** | Problem input with category tags. Submits to Claude API, generates full JSON roadmap |
| **My Roadmap** | Path diagram (SVG node graph), guidance text, action plan, AI chat, decision tree |
| **Community** | Filterable story cards with likes |
| **Skills** | Click any skill → full-screen visual framework (learning ladder, timeline, job titles, work paths, earnings, prerequisites) |

Every diagram is **generated by Claude** from structured JSON — no static images, no hardcoded data.

---

## Tech Stack

```
React 18          — single JSX file, no router needed
Anthropic API     — claude-sonnet-4-20250514 via fetch()
Vite              — dev server + build (or use CRA / any bundler)
Google Fonts      — Clash Display, Cabinet Grotesk, Instrument Serif (CDN, no install)
SVG               — all diagrams are inline SVG, no chart library deps
```

Zero npm dependencies beyond React and a bundler. No Tailwind. No component library. All styles are inline CSS-in-JS inside the `CSS` const.

## Setup

### 1. Prerequisites

- Node.js 18+ ([nodejs.org](https://nodejs.org))
- An Anthropic API key ([console.anthropic.com](https://console.anthropic.com))

### 2. Create the project

```bash
npm create vite@latest lifepath -- --template react
cd lifepath
```

### 3. Drop in the app file

Replace `src/App.jsx` with `lifepath-v3.jsx` (rename it to `App.jsx`):

```bash
cp lifepath-v3.jsx lifepath/src/App.jsx
```

### 4. Set up your API key

Create a `.env` file in the project root:

```bash
# lifepath/.env
VITE_ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Then update the `callAI` function in `App.jsx` to use it. Find this block (around line 400):

```js
const r = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify({...}),
});
```

Change it to:

```js
const r = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": import.meta.env.VITE_ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01",
    "anthropic-dangerous-direct-browser-access": "true",
  },
  body: JSON.stringify({...}),
});
```

> **Note:** Direct browser → Anthropic API calls require the `anthropic-dangerous-direct-browser-access: true` header. This is fine for local dev and prototypes. For production, route API calls through your own backend to keep the key server-side.

### 5. Update `src/main.jsx`

Make sure it points to App:

```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
```

### 6. Update `index.html`

Make sure the root div exists:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>LifePath</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>
```

### 7. Install and run

```bash
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173)

## Build for Production


Output goes to `dist/`. Deploy to Vercel, Netlify, or any static host.

```bash

npm i -g vercel
```

Remember to add `VITE_ANTHROPIC_API_KEY` as an environment variable in your hosting dashboard. For production, strongly consider moving the API call to a serverless function (Vercel Edge Function, Netlify Function, etc.) so the key is never exposed in the browser bundle.

## How the AI Works

All AI calls go through one function:

```js
async function callAI(messages, system) {
  // calls claude-sonnet-4-20250514
  // returns raw text string
}
```

There are three places it's called:

| Caller | What it asks for | Returns |
|---|---|---|
| `HeroPage` | Full roadmap JSON (summary, steps, skills, resources, diagram nodes) | Structured JSON |
| `AIChatWidget` | Conversational life coach response | Plain text |
| `SkillsPage` | Skill framework JSON (ladder, timeline, job titles, earnings, prereqs) | Structured JSON |

Both JSON responses have fallback data so the UI always renders even if parsing fails.

## Customisation !

**Add a new skill card:**

Find the `SKILLS` array and add an entry:

```js
{ name:"Your Skill", ico:"bolt", demand:85, color:"#your-hex", lv:"Hot" }
```

**Add a skill palette** (so it gets its own color scheme in the framework diagram):

Find `SKILL_PALETTES` and add:

```js
"Your Skill": { bg:"#...", panel:"#...", a:"#accent", b:"#accent2", text:"#..." }
```

**Add a community story:**

Find the `STORIES` array and add an object matching the existing shape.

**Change the AI model:**

In `callAI`, change `"claude-sonnet-4-20250514"` to any model available on your API plan.

## Known Limitations

- **No backend / database** — all state is in-memory. Refresh loses roadmap. Add a backend (Supabase, Firebase, PlanetScale) to persist user sessions.
- **API key in browser** — fine for dev/prototypes, not for production. Move to a backend route.
- **Community stories are static** — hardcoded in `STORIES`. Connect to a real database to make them user-submitted.
- **Feedback form is UI-only** — it simulates a submit delay but doesn't send anywhere. Wire up to an email service (Resend, Postmark) or a form backend (Formspree, Airtable) to receive responses.
- **No auth** — there's no login system. Add Clerk, Supabase Auth, or NextAuth if you want user accounts.

## Recommended Next Steps

If you want to turn this into a real product:

1. **Backend:** Next.js App Router or Hono on Cloudflare Workers
2. **Database:** Supabase (Postgres + Auth + Storage in one)
3. **API proxy:** Move `callAI` to a server route — never expose keys in the browser
4. **Persistence:** Save roadmaps to DB, let users return to their plan
5. **Auth:** Supabase Auth or Clerk for user accounts
6. **Analytics:** PostHog (open source, self-hostable)
7. **Deployment:** Vercel for the frontend, Supabase for data
